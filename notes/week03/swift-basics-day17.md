# Swift Basics Day 17

## 今天学了什么
- 简单数据模型设计
- 为什么很多业务数据适合先用 struct
- 字段类型要贴近真实含义
- 模型嵌套模型
- 模型可以包含数组属性
- 模型可以有轻量计算属性
- 多个小模型可以组合成更像业务对象的大模型

## 核心理解
- 建模不是把字段凑齐，而是把信息表达清楚
- age / views / likes / isPublished 这些字段不应该乱写成 String
- 作者、统计信息这类东西本身也可以是独立模型
- 真实业务对象通常是多个小模型组合起来的
- 模型里可以放轻量展示逻辑，但不要塞太重的业务逻辑

## 今天最重要的收获
- 我开始不再把“数据”看成一堆散乱字段
- 我开始尝试把业务信息组织成有结构的模型
- VideoDetail 这种组合模型已经有一点真实业务对象的感觉了

## 以后可以继续优化的方向
- 字段命名更具体
- duration 这类字段标明单位
- level 这类字段用更明确的业务语义
